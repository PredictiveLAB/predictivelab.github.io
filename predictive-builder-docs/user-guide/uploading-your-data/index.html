Data cleaining & labeling

La opción de cargado y etiquetado de datos da paso al usuario a que elija entre, “Crear reporte” o “Crear Dataset”. A continuación, se detallan las posibilidades de ambas opciones:

Create a Dataset

La creación de dataset corresponde al proceso de inicializar una nueva base de datos, los cuales debe rellenar las siguientes casillas mostradas en la siguiente figura.

  • Nombre: Este es el nombre (Los nombres se pueden repetir en la aplicación, por lo tanto, se recomienda tener diferentes nombres al momento de subir cada archivo para no llevar a confusión) por el cual será referido el dataset en aplicación.
  • Tipo de equipo: Despliega una barra deslizante que muestra opciones de equipos a elegir. Esto sirve para 3 acciones principalmente:
    • Para que agregar más información al dataset, incluyendo el tipo de sensores y que miden.
    • Al momento de entrenar se puede seleccionar cualquier modelo que haya sido entrenado con ese tipo de equipo.
    • Ordena los equipos según el tipo en la sección estadísticas.

  • Es pública: En caso de que esta casilla sea seleccionada, se accede a que el dataset pueda ser utilizada en otras sesiones de la aplicación, sin al necesitad de solicitar autorización al usuario para ello.

  • Subir datos: Si se hace clic a esta casilla, se abrirá un explorador de archivos para seleccionar un archivo que obtenga el dataset en estudio, estos podrían tener la extensión “.txt”, “. xlxs” y “.csv, la aplicación lo detectará automáticamente.

    • Los archivos subidos a la aplicación deben tener una cierta forma para poder ser admitidos y hacer un correcto análisis del equipo. En síntesis, estos deben poseer una serie de tiempo y un label, los cuales están en las columnas “timestamp” y “target”, respectivamente.

Al momento subir un nuevo dataset, se abrirá una sección de etiquetado, inmediatamente debajo de subir datos que puede ser observada en la siguiente figura.

Lo que esta sección hace es visualizar el etiquetado que se tenga en una serie de tiempo y elegir la correcta etiqueta. Si el proceso anterior es exitoso, se visualizará: - El porcentaje de Nans por columnas. - Análisis de correlación entre sensores - Posibilidad de hacer la transformada de Fourier, variando los parámetros en tiempo real - Análisis de parámetros - Eliminar los sensores que sean pertinentes para el usuario Todo lo anterior con el propósito de eliminar sensores.

Create a Report

La primera opción corresponde a subir un archivo con extensión .csv para ser guardado en la aplicación. El dataset será guardado para poder predecir su estado de salud, es decir este archivo no posee etiqueta. En la siguiente figura se pueden observar los parámetros necesarios que consisten en:

  • Información del reporte: Se debe elegir si te subirá un reporte simple o una con una columna de serie temporal, la diferencia radica en que el primero no utiliza serie temporal. En el primer caso solo se requerirá el periodo de tiempo inicial y final, en cambio en el segundo se pedirá la misma información y el periodo de muestreo en segundos.
  • Elija un archivo de tu lista: Se debe elegir un activo de la lista de equipos monitoreados.
  • Mediciones cargadas: Se sube el archivo con los datos. Luego, se hace clic a “enviar” y el archivo será guardado dentro de la aplicación. Es importante destacar que si se sube una serie de tiempo, la columna con el tiempo debe ser llamada “timestamp”.
