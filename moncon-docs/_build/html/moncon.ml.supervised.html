predict(x, timestamps, aggregation, log_filepath)[source]

Abstract method for all ML architectures use their internal model to predict on data. Returns the predicted values and the result of their aggregation (using ‘aggregate’ method). Timestamps are only used while logging.

Args:

x (np.ndarray): Input data timestamps (pd.Series): Timestamps of the input data aggregation (str): Aggregation method to be used for prediction log_filepath (str): Path to the log file used to log prediction progress

Returns:

Tuple[np.ndarray, np.ndarray]: Tuple with predicted values and aggregated values

class moncon.ml.supervised.models.CNN(arch_params, chckpt_filepath=None)[source]

Bases: DLSupervisedModel

Classification model based on a CNN architecture. Inherits from DLSupervisedModel

Redefines the following methods:

  • init_arch

  • parse_fit_logging

  • parse_predict_logging

Requires the following values in arch_params:

  • arch_params[“time_size”] (int): Number of time steps in the input to use in each window.

  • arch_params[“overlap”] (int): Number of time steps to overlap between windows.

  • arch_params[“feature_size”] (int): number of input features.

  • arch_params[“filter_num”] (int): number of filters in the first CNN block (it is doubled in each block)

  • arch_params[“kernel_v”] (int): vertical kernel size

  • arch_params[“kernel_h”] (int): horizontal kernel size

  • arch_params[“out_size”] (int): The number of output classes.

  • arch_params[“metric”] (str): The metric to use.

Requires the following values in train_params (for fit method):

  • train_params[“batch_size”] (int): Batch size for training

  • train_params[“epochs”] (int): Number of epochs for training
